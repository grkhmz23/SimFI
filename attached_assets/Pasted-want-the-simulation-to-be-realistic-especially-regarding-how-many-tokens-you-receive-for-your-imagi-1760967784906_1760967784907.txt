The "-100% loss" problem arises because the simple tokens = sol / price calculation ignores how the bonding curve works: the price increases as you buy. Buying 1 SOL worth might give you far fewer tokens than expected if you only use the initial spot price.

The Best Approach for Realistic Paper Trading:

Even for paper trading, the most educational and realistic way is to simulate the trade using a quoting mechanism that understands bonding curves and liquidity pools.

Use the Jupiter V6 Quote API (Recommended):

Keep the Backend Quote Endpoint: Implement the /api/tokens/quote/buy endpoint in your backend as described in the previous message. This endpoint takes the token address and the imaginary SOL amount the user wants to spend.

Call Jupiter: Your backend calls the actual Jupiter V6 Quote API (https://quote-api.jup.ag/v6/quote). Jupiter calculates how many tokens the user would receive (outAmount) for their SOL amount, considering the current state of the liquidity pool or bonding curve (if Jupiter indexes pump.fun pools directly or via Raydium pools after migration). It also gives you the effectivePrice (average price paid).

Update Frontend (TradeModal.js):

Change the main input field to ask the user "SOL to Spend" (e.g., default to 1).

When the user enters a SOL amount, call your backend /api/tokens/quote/buy endpoint.

Display the estimated tokenAmountOut (from the quote) that they would receive. Clearly label this (e.g., "Estimated Tokens Received:").

You can also display the effectivePriceSol from the quote.

Execute the Paper Trade:

When the user clicks "Buy Now" in the modal:

Get the solAmount (user input), tokenAmountOutLamports (from the quote), and effectivePriceSol (from the quote).

Call your existing backend /api/trades/buy endpoint.

Crucially: Pass the solAmount as sol_spent, the tokenAmountOutLamports (converted appropriately if needed) as the amount of tokens, and the effectivePriceSol as the entry_price.

Record the Position: Your backend /api/trades/buy endpoint simply records this simulated trade in your database (deducts imaginary SOL balance, adds the imaginary token position) using the numbers obtained from the Jupiter quote. No actual blockchain transaction happens.
